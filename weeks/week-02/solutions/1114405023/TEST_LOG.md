###Task1 — sequence_clean
#Red（測試失敗階段）
首先建立測試檔案 tests/test_task1.py，並使用 Python 內建的 unittest 撰寫測試骨架。
在測試檔中設計了三個測試案例：
正常情況（normal case）
空列表（empty list）
全部為重複元素（all duplicates）
在開發初期，sequence_clean() 函式尚未完整實作，因此測試無法通過。
在實作過程中曾出現幾個問題：
函式名稱一開始不正確。
不小心重複定義多個同名函式。
函式只回傳一個結果，而題目需要回傳四個結果。
測試檔中的縮排錯誤，導致其中一個測試沒有被偵測到。
因此在初期測試是失敗的。

#Green（測試成功階段）
修正 task1_sequence_clean.py 中的程式後，完成 sequence_clean() 函式。
該函式包含四個功能：
去除重複元素並保留第一次出現的順序
將原列表進行升序排序
將原列表進行降序排序
擷取所有偶數並保留原本順序
同時修正測試檔的結構，使三個測試案例都能正確執行。

#Refactor（程式重構）
在測試通過後，對程式進行簡單的重構以提升可讀性：
使用清楚的變數名稱，例如 dedupe、asc、desc、evens
使用 Python 內建函式 sorted() 簡化排序程式
將測試案例整理在同一個測試類別中，使結構更清楚
重構後再次執行測試，所有測試仍然通過。

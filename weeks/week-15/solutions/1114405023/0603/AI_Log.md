# AI_LOG.md

## 作業名稱

6/3 Starter — UVA 11417 GCD

## 使用 AI 的目的

本次作業使用 AI 協助理解 UVA 11417 GCD 題目需求、設計測試案例、確認 TDD 流程，以及檢查測試結果是否符合紅燈與綠燈階段。

## 與 AI 討論的內容

1. 確認題目需求：

   * `sum_of_gcd(n)` 需要計算 `1 <= i < j <= n` 範圍內所有 `gcd(i, j)` 的總和。
   * 例如 `n = 2` 時，只有 `gcd(1, 2) = 1`，所以答案是 `1`。
   * 例如 `n = 10` 時，題目範例答案是 `67`。
   * `n = 1` 時，沒有任何符合 `1 <= i < j <= n` 的組合，所以答案是 `0`。

2. 設計測試案例：

   * 基本案例：`sum_of_gcd(2)` 應回傳 `1`。
   * 範例案例：`sum_of_gcd(10)` 應回傳 `67`。
   * 邊界案例：`sum_of_gcd(1)` 應回傳 `0`。
   * 額外案例：`sum_of_gcd(3)` 應回傳 `3`。

3. 確認 TDD 紅燈：

   * 一開始測試檔案引用 `from gcd import sum_of_gcd`。
   * 由於當時尚未建立 `gcd.py`，執行測試時出現 `ModuleNotFoundError: No module named 'gcd'`。
   * AI 說明這屬於 TDD 流程中的紅燈，代表測試已經開始檢查尚未實作的功能。

4. 確認 TDD 綠燈：

   * 建立 `gcd.py` 並完成 `sum_of_gcd(n)` 實作後，再次執行測試。
   * 測試結果顯示 `Ran 4 tests` 與 `OK`，代表所有測試通過。

## 我採用的 AI 建議

我採用了 AI 建議的測試案例設計，包含基本案例、題目範例、邊界案例與額外驗證案例。這些測試可以確認 `sum_of_gcd(n)` 是否正確計算所有 `gcd(i, j)` 的總和。
